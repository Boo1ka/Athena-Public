## 3. Execution Protocol (At `/end`)

### Step 1: Session Scan

Review the session log for new patterns, insights, or frameworks discovered.

**Detection Questions**:

1. Did we articulate a new structural pattern?
2. Did we solve a problem in one domain that applies elsewhere?
3. Did we identify a failure mode that has parallels?

### Step 2: Cross-Domain Transfer

For each pattern detected, ask:

```text
┌─────────────────────────────────────────────────────────────────────────────────┐
│  CROSS-POLLINATION QUESTIONS                                                    │
├─────────────────────────────────────────────────────────────────────────────────┤
│  1. "Where else does this structure appear?"                                    │
│  2. "What's the equivalent failure mode in [other domain]?"                     │
│  3. "Can this framework be named and filed as a protocol?"                      │
│  4. "Does this connect to an existing protocol via isomorphism?"                │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### Step 3: Documentation

| If | Then |
|----|------|
| New isomorphism found | Add to Isomorphism Matrix in this file |
| New protocol warranted | Create `XX-protocol-name.md` + register in SKILL_INDEX |
| Existing protocol gains new domain | Update existing protocol + add `#isomorphism` tag |
| Cross-reference discovered | Update TAG_INDEX via `generate_tag_index.py` |

### Step 4: Session Log Entry

Append to session log under "Key Insights":

```markdown
### Cross-Pollination (Protocol 67)
- **Pattern**: [Name]
- **Source Domain**: [Where it was discovered]
- **Transfer Domains**: [Where else it applies]
- **Action**: [Protocol created / Updated / Documented]
```
